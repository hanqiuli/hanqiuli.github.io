# Why Simple Websites Win

The web has become bloated. Pages that could be 10KB are now 10MB. Sites that could load instantly now take seconds.

## The Problem

Most websites don't need:
- JavaScript frameworks
- Build systems
- npm packages
- CI/CD pipelines

For a personal site or blog, all you need is HTML and CSS.

## The Solution

Write HTML. Write CSS. Upload files. Done.

No build step. No dependencies. No security vulnerabilities from outdated packages. Just content.

## Performance

A simple HTML page loads in milliseconds. It works on any device, any connection, any browser. It will still work in 20 years.

Can you say the same about your React app?

## Longevity

The sites I admire most are the ones that haven't changed in years. They still work. They still load fast. They still look good.

Simple wins.
